# What are agent skills

It's an open format accepted across coding agents, including Claude Code. Agent skills are folders of instructions, scripts, and resources that agents can discover.

This is the idea behind skills: **they allow you to progressively disclose instructions.**

The agent will be able to see all the skills title and description and use it when needed. 

The skills can be scoped to the project and to the user, like the `agent.md` files. 

_A skill can also refer to other file._

## Types of skills

There are 2 types of skills. The ones you want the LLM to invoke and the want **you** want to trigger (used to be custom slash command). 

You can make sure the skill cannot be invoked by the LLM with: `disable-model-invocation: true`. You can make sure the user cannot invoke if with `user-invocable: false`.

## Arguments

- You can pass arguments and set the argument-hint in the frontmatter. 

See the [skills documentation](https://code.claude.com/docs/en/skills) for information about what can be set in the skill.

```md
---
description: Create a UI component
allowed-tools: Read, Write, Edit, Glob, Bash(pnpm test:*)
argument-hint: "[Component name]"
---

Component name: $ARGUMENTS
```

You can also access specific arguments by index using `$ARGUMENTS[N]` (0-based):

```md
---
description: Create a UI component with a variant
allowed-tools: Read, Write, Edit, Glob, Bash(pnpm test:*)
argument-hint: "[ComponentName] [variant]"
---

Create a new UI component named $ARGUMENTS[0] with the "$ARGUMENTS[1]" variant (e.g. "primary", "ghost", "destructive").
```

Invoked as `/create-component Button ghost`, `$ARGUMENTS[0]` is `Button` and `$ARGUMENTS[1]` is `ghost`.
