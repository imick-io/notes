# Introduction

First, you’ll need a Claude plan. The free version works, but the results won’t be as strong.
Next, install Claude Code. The best approach is to follow the official Claude documentation for setup.
After that, run `claude` in your terminal and answer the prompts you’re given.
If you’re having trouble installing Claude Code, it may be worth taking a basic programming course to build the necessary foundations.
It’s also often better to establish a solid foundation before asking the AI to build features. The model will base its output on your existing code, architecture, and patterns, so setting clear structure and principles early leads to much better results in my experience. 

## Terminal & IDE
You can use the integrated terminal in your IDE, which is usually set up by default. If it isn’t, you can link your IDE and Claude Code using the `/ide` command.

## A Basic Workflow with Git
A good habit is to use Claude Code alongside Git for version control. A simple workflow looks like this: commit your current work before asking Claude to make changes, then review the diff of what it produced. If the changes look right, commit them. If not, you can easily revert. Claude Code runs in an interactive mode where you’ll be asked to approve file edits and certain shell commands before they execute, so you stay in control at each step. Pairing that with Git gives you a safety net to experiment freely. If you’re not familiar with Git, check out my separate Git class to get up to speed.

## Conversation History
Claude Code saves your sessions so you can pick up where you left off.

- **Continue the most recent session:** `claude --continue` (or `-c`)
- **Resume a past session from a list:** `claude --resume` (or `-r`)
- **Rename the current session:** type `/rename <name>` during a conversation
- **Start fresh:** just run `claude` — this always opens a new session

## Choosing a Model
The Claude model lineup changes regularly, so it's worth staying familiar with what's available. As a rule of thumb, match the model to the task: use a smaller, faster model for routine edits and quick questions, and reach for the most capable model when the problem is complex or the stakes are high. The goal is to get the right result with the least amount of tokens spent — which keeps things fast and cost-efficient.

## Usage
There are two ways to keep an eye on your usage. Inside a session, type `/usage` to see a summary of tokens consumed in the current conversation. For a broader view — billing, monthly totals, and usage across all sessions — log in to your account at claude.ai or the Anthropic Console at console.anthropic.com, where you'll find a dedicated usage dashboard.


### Always-Visible Usage

By default, Claude Code only shows context usage on demand via `/usage`. To display it persistently in the terminal status line, use the `ccstatusline` tool.

**Step 1 — Configure ccstatusline**

Create the file `~/.config/ccstatusline/settings.json` and add the following. `~/.config` is the standard directory on macOS and Linux where user-specific application configuration files are stored — it lives in your home folder and is separate from any project. Placing the config here makes it global, so it applies to every Claude Code session regardless of project.

```json
{
  "version": 3,
  "lines": [
    [
      {
        "id": "1",
        "type": "context-length",
        "color": "yellow",
        "bold": true,
        "rawValue": true
      },
      {
        "id": "2",
        "type": "custom-text",
        "customText": "(",
        "color": "brightBlack",
        "merge": "no-padding"
      },
      {
        "id": "3",
        "type": "context-percentage",
        "color": "brightBlack",
        "rawValue": true,
        "merge": "no-padding"
      },
      {
        "id": "4",
        "type": "custom-text",
        "customText": ")",
        "color": "brightBlack",
        "merge": "no-padding"
      }
    ],
    [],
    []
  ],
  "flexMode": "full-minus-40",
  "compactThreshold": 60,
  "colorLevel": 2,
  "defaultSeparator": " ",
  "inheritSeparatorColors": false,
  "globalBold": false,
  "powerline": {
    "enabled": false,
    "separators": [" "],
    "separatorInvertBackground": [false],
    "startCaps": [],
    "endCaps": [],
    "autoAlign": false
  }
}
```

**Step 2 — Tell Claude Code to use it**

Add the following to `~/.claude/settings.json`:

```json
{
  "statusLine": {
    "type": "command",
    "command": "npx ccstatusline@latest"
  }
}
```

Once both files are saved, the context length and percentage will appear in the status line every time you run Claude Code.
