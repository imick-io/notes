# Introduction

Software quality is now very important. AI Agent are super sensitive to good/bad codebase. 

A good codebase is easy to change. 

Software Entropy: The more the codebase becomes bigger, the more the codebase becomes bad. 

AI Agent are not good at preventing software entropy. That's why it is the responsability of the developer to dictate good coding practice, refactor when needed. AI is even a software entropy accelerator. The AI Agent will get inspired a lot by the codebase. So, when a codebase is bad, it will generate bad code since it gets inspired by it a lot. 

So, the goal is to provide feedback loops to the agent to minimize software entropy. Like the human developer, he will test and validate frequently before working on the other task. 

A good feedback is of course using a strongly typed language, adding automated tests...

Another good feedback loop to add is formatting on `pre-commit`.

1. Plan
2. Implement (TDD)
3. Feedback
4. Commit
